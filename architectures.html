Transformers

The Transformer architecture, introduced in "Attention Is All You Need" (2017), revolutionized AI and forms the backbone of modern multimodal systems.

Transformer Architecture
Multi-Head Attention
Add & Norm
Feed Forward
Add & Norm

Key Components

Self-Attention

Allows each position to attend to all other positions, capturing long-range dependencies.

Multi-Head Attention

Multiple attention heads learn different relationship patterns simultaneously.

Positional Encoding

Injects sequence order information since attention is permutation-invariant.

Feed-Forward Networks

Position-wise fully connected layers for non-linear transformations.

Why Transformers for Multimodal?

  • Modality Agnostic: Same architecture works for text, images, audio
  • Cross-Attention: Natural way to relate different modalities
  • Scalability: Performance improves with more data and compute
  • Parallelization: Efficient training on modern hardware

CLIP (Contrastive Language-Image Pre-training)

CLIP by OpenAI learns visual concepts from natural language supervision, creating aligned text and image representations.

How CLIP Works

  1. Encode images and text captions into embeddings
  2. Compute similarity between all image-text pairs in a batch
  3. Train with contrastive loss: maximize similarity of matching pairs
  4. Result: aligned embedding space for both modalities

Vision Transformers (ViT)

ViT applies the Transformer architecture directly to images by treating image patches as tokens, achieving state-of-the-art results.

Key Innovations

  • Patch Embedding: Split image into fixed-size patches (e.g., 16x16)
  • Linear Projection: Flatten patches and project to embedding dimension
  • [CLS] Token: Special token for image-level representation
  • Position Embeddings: Learnable or sinusoidal positional encodings
